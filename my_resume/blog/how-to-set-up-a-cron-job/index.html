How to Set Up a Cron Job

Anam Ibn Jafar 8th July, 2025

What is Cron?

Cron is a time-based job scheduler in Unix/Linux systems. It lets you run scripts or commands automatically at specified intervals — every minute, every hour, every day, or any custom schedule. It's perfect for automating repetitive tasks like backups, log cleanup, data fetching, and more.

Cron Syntax

A cron expression has 5 fields:

┌───────────── minute (0 - 59)
│ ┌───────────── hour (0 - 23)
│ │ ┌───────────── day of month (1 - 31)
│ │ │ ┌───────────── month (1 - 12)
│ │ │ │ ┌───────────── day of week (0 - 7, 0 and 7 = Sunday)
│ │ │ │ │
* * * * *  command_to_run

Examples:

  • 0 9 * * * — Every day at 9:00 AM
  • */5 * * * * — Every 5 minutes
  • 0 0 * * 0 — Every Sunday at midnight
  • 30 6 1 * * — 6:30 AM on the 1st of every month

Real Example: Daily Weather Logger

Let's build a cron job that fetches the weather every morning at 7:00 AM and writes it to a log file. We'll use the free wttr.in API.

Step 1: Create the Script

mkdir -p ~/scripts
nano ~/scripts/weather.sh

Add the following:

#!/bin/bash

# Weather logger script
LOG_FILE="$HOME/weather_log.txt"
CITY="Dhaka"

# Fetch weather
WEATHER=$(curl -s "wttr.in/$CITY?format=%C+%t+%h+%w")

# Get current date and time
TIMESTAMP=$(date '+%Y-%m-%d %H:%M:%S')

# Write to log
echo "[$TIMESTAMP] $CITY: $WEATHER" >> "$LOG_FILE"

Step 2: Make It Executable

chmod +x ~/scripts/weather.sh

Step 3: Test It

# Run it manually first
./scripts/weather.sh

# Check the output
cat ~/weather_log.txt
# [2025-07-08 07:00:00] Dhaka: Partly cloudy +32°C 78% ↑11km/h

Step 4: Set Up the Cron Job

# Open the crontab editor
crontab -e

# Add this line at the bottom:
0 7 * * * /home/your-username/scripts/weather.sh

This runs the script every day at 7:00 AM. The weather info gets appended to ~/weather_log.txt.

Step 5: Verify

# List your cron jobs
crontab -l

# Check cron logs (Ubuntu)
grep CRON /var/log/syslog | tail -5

Common Gotchas

  • Cron uses a minimal environment — always use full paths for commands (e.g., /usr/bin/curl instead of just curl)
  • Redirect errors to see what went wrong: 0 7 * * * /home/user/scripts/weather.sh 2>&1 >> /home/user/cron.log
  • Make sure the script is executable (chmod +x)

That's it! You now have an automated weather logger. Apply the same pattern to any task — database backups, log rotation, API polling, deployment scripts — the possibilities are endless.
