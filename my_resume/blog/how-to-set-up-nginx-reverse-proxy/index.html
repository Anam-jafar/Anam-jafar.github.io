How to Set Up Nginx Reverse Proxy

Anam Ibn Jafar 25th June, 2025

What is Nginx?

Nginx (pronounced "engine-x") is a high-performance web server that can also act as a reverse proxy, load balancer, and HTTP cache. It's lightweight, fast, and handles thousands of concurrent connections efficiently.

Why Do You Need a Reverse Proxy?

Imagine this: you have a single server running 3 different projects — a Laravel app on port 8000, a FastAPI service on port 8001, and a React app on port 3000. But your server only exposes port 80 (HTTP) and 443 (HTTPS) to the outside world.

How do you serve all three? You can't tell users to visit yoursite.com:8001. This is where Nginx reverse proxy comes in — it listens on port 80 and routes requests to the correct app based on the URL path.

flowchart LR
    Client["🌐 Client Browser"]
    Nginx["⚡ Nginx\nReverse Proxy\nPort 80"]
    App1["🟧 Laravel App\n127.0.0.1:8000"]
    App2["🟦 FastAPI Service\n127.0.0.1:8001"]
    App3["🟩 React Dashboard\n127.0.0.1:3000"]

    Client -- "yoursite.com/*" --> Nginx
    Nginx -- "location /" --> App1
    Nginx -- "location /api/" --> App2
    Nginx -- "location /dashboard/" --> App3

Step 1: Install Nginx

sudo apt update
sudo apt install nginx -y
sudo systemctl enable nginx
sudo systemctl start nginx

Step 2: Configure Location Blocks

Edit the Nginx config to define a location block for each project. Each prefix route maps to a different backend service.

sudo nano /etc/nginx/sites-available/default

Replace the contents with:

server {
    listen 80;
    server_name yoursite.com;

    # Project 1 — Laravel App
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Project 2 — FastAPI Service
    location /api/ {
        proxy_pass http://127.0.0.1:8001/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # Project 3 — React Dashboard
    location /dashboard/ {
        proxy_pass http://127.0.0.1:3000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

How It Works

  • yoursite.com/ → hits the Laravel app on port 8000
  • yoursite.com/api/users → hits the FastAPI service on port 8001 at /users
  • yoursite.com/dashboard/ → hits the React app on port 3000

Notice the trailing slash in proxy_pass — this strips the prefix. So /api/users gets forwarded as /users to the FastAPI backend.

Request Routing Decision Flow

flowchart TD
    Req["Incoming Request\nyoursite.com/api/users"]
    Match{{"Nginx matches\nlocation blocks\n(longest prefix wins)"}}
    L1["/api/ ✅ matches"]
    L2["/ ✅ matches"]
    L3["/dashboard/ ❌ no match"]
    Winner["Winner: /api/\n(longest prefix)"]
    Strip["Strip prefix /api/\n→ forwarded as /users"]
    Backend["FastAPI receives\nGET /users on :8001"]

    Req --> Match
    Match --> L1
    Match --> L2
    Match --> L3
    L1 --> Winner
    L2 -.->|"shorter prefix,\nlower priority"| Winner
    Winner --> Strip
    Strip --> Backend

Step 3: Test and Reload

# Test config for syntax errors
sudo nginx -t

# Reload Nginx
sudo systemctl reload nginx

Bonus: Add SSL with Certbot

sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d yoursite.com

Certbot will automatically configure HTTPS and set up auto-renewal. Now all your projects are served securely through a single entry point.

With SSL — The Complete Picture

flowchart LR
    Client["🌐 Client"]
    SSL["🔒 SSL/TLS\nTermination\nPort 443"]
    Redirect["Port 80\n→ 301 Redirect"]
    Nginx["⚡ Nginx"]
    App1["Laravel\n:8000"]
    App2["FastAPI\n:8001"]
    App3["React\n:3000"]

    Client -- "HTTPS" --> SSL
    Client -. "HTTP" .-> Redirect
    Redirect -. "→ HTTPS" .-> SSL
    SSL --> Nginx
    Nginx -- "plain HTTP" --> App1
    Nginx -- "plain HTTP" --> App2
    Nginx -- "plain HTTP" --> App3

Nginx handles SSL termination — meaning encrypted traffic from clients is decrypted at Nginx, then forwarded to your backend apps as plain HTTP over localhost. Your apps never need to deal with certificates.
