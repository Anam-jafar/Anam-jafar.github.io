Laravel PHP Queues

Background Processing in Laravel

Anam Ibn Jafar 20th July, 2025

The Problem

Your user clicks "Place Order" and your app needs to send a confirmation email, generate a PDF invoice, notify the admin, and update analytics. If you do all this synchronously in the controller, the user stares at a loading spinner for 10+ seconds. That's a terrible experience.

The Solution: Queues & Jobs

Laravel's queue system lets you defer time-consuming tasks to be processed in the background. The user gets an instant response, and the heavy work happens behind the scenes.

Step 1: Configure Queue Driver

In your .env file, set the queue driver. For production, use Redis or database. For local testing, database is simplest:

QUEUE_CONNECTION=database

Create the jobs table:

php artisan queue:table
php artisan migrate

Step 2: Create a Job

php artisan make:job SendOrderConfirmation

This creates app/Jobs/SendOrderConfirmation.php:

<?php

namespace App\Jobs;

use App\Models\Order;
use App\Mail\OrderConfirmationMail;
use Illuminate\Bus\Queueable;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Foundation\Bus\Dispatchable;
use Illuminate\Queue\InteractsWithQueue;
use Illuminate\Queue\SerializesModels;
use Illuminate\Support\Facades\Mail;

class SendOrderConfirmation implements ShouldQueue
{
    use Dispatchable, InteractsWithQueue, Queueable, SerializesModels;

    public function __construct(
        public Order $order
    ) {}

    public function handle(): void
    {
        Mail::to($this->order->user->email)
            ->send(new OrderConfirmationMail($this->order));
    }

    public int $tries = 3;

    public function backoff(): array
    {
        return [30, 60, 120];
    }
}

Step 3: Dispatch the Job

In your controller, dispatch the job instead of doing the work inline:

// Before (slow - user waits)
Mail::to($order->user->email)->send(new OrderConfirmationMail($order));

// After (fast - user gets instant response)
SendOrderConfirmation::dispatch($order);

Step 4: Run the Queue Worker

# Start processing queued jobs
php artisan queue:work

# For production, use with retry and timeout
php artisan queue:work --tries=3 --timeout=90

Job Chaining

Need multiple jobs to run in sequence? Chain them:

use Illuminate\Support\Facades\Bus;

Bus::chain([
    new SendOrderConfirmation($order),
    new GenerateInvoicePdf($order),
    new NotifyAdmin($order),
])->dispatch();

Delayed Jobs

Need to run a job after a delay?

// Send a follow-up email 24 hours later
SendFollowUpEmail::dispatch($order)
    ->delay(now()->addHours(24));

Monitoring Failed Jobs

# Create the failed jobs table
php artisan queue:failed-table
php artisan migrate

# View failed jobs
php artisan queue:failed

# Retry a specific failed job
php artisan queue:retry <job-id>

# Retry all failed jobs
php artisan queue:retry all

Production Tip: Use Supervisor

In production, use Supervisor to keep the queue worker running and auto-restart it if it crashes:

[program:laravel-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/your-app/artisan queue:work --sleep=3 --tries=3 --max-time=3600
autostart=true
autorestart=true
stopasgroup=true
killasgroup=true
numprocs=2
redirect_stderr=true
stdout_logfile=/var/log/worker.log

Background processing is one of the most impactful patterns in Laravel. It keeps your app fast, your users happy, and your server resources well-utilized.
